## 📋 需求概述

### 问题描述
用户发送 `/new`、`/reset`、`/clear`、`新会话`、`重新开始`、`清空对话` 等命令时：
- ❌ 未创建新会话
- ❌ 未清空对话上下文
- ✅ 命令被 normalizeSlashCommand 统一转换为 `/new` 并发送给 Gateway

### 根本原因
当前代码中 `sessionKey` 基于 `SessionContext` 生成，对于同一对话是固定不变的。Gateway 收到 `userContent="/new"` 时，由于 `sessionKey` 不变，无法识别需要创建新会话。

### 修复方案（方案1）
检测到新会话命令时，在 `SessionContext` 中注入 `timestamp` 字段，生成唯一的 `sessionKey`，强制 Gateway 创建新会话。

**文件**：`plugin.ts`

**修改内容**：

   ```typescript
   const rawText = content.text || '';
   const normalizedText = normalizeSlashCommand(rawText);
   const isNewSessionCommand = normalizedText === '/new';
   ```

2. 构建基础 SessionContext：
   ```typescript
   const sessionContext = buildSessionContext({
     accountId,
     senderId,
     senderName,
     conversationType: data.conversationType,
     conversationId: data.conversationId,
     groupSubject: data.conversationTitle,
     separateSessionByConversation,
     groupSessionScope,
   });
   ```

3. 注入时间戳（如果是新会话命令）：
   ```typescript
   // 检测新会话命令，在 sessionKey 中注入时间戳强制创建新会话
   const finalSessionContext = isNewSessionCommand
     ? { ...sessionContext, timestamp: Date.now() }
     : sessionContext;

   const sessionKey = JSON.stringify(finalSessionContext);
   ```

4. 调整 userContent（可选优化）：
